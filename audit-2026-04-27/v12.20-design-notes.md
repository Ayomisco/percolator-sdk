## c175ec4 — deposits-only insurance mode

Adds:
- New MarketConfig field `insurance_withdraw_deposits_only: u8`
  (claimed from existing `_iw_padding`). Layout-compatible: same total
  size, but a previously-zero byte now carries semantic meaning.
- New wire-format flag `INSURANCE_WITHDRAW_DEPOSITS_ONLY_FLAG = 0x8000`
  packed into the `insurance_withdraw_max_bps: u16` argument of
  `InitMarket`. Mask `INSURANCE_WITHDRAW_MAX_BPS_MASK = 0x7FFF`.
- New behavior in `WithdrawInsuranceLimited` (tag 23): when the flag
  is set, only TopUpInsurance principal can be withdrawn.

### SDK changes required
1. `encodeInitMarket` v12.20 target. Either:
   - Add a separate `insuranceWithdrawDepositsOnly?: boolean` argument
     and pack it into `insuranceWithdrawMaxBps` at encode time.
   - Or add a `Target` enum value `'v12.20'` and gate the flag.
2. `parseConfig` (`src/solana/slab.ts`) needs to expose the new field
   when the slab was created on v12.20+. Reading the existing byte from
   v12.17/v12.19 slabs will return 0 (correct default).
3. SDK type `InitMarketExtendedTail` may need a `depositsOnlyMode`
   convenience field.

### Migration plan
- Single PR `feat(sdk): v12.20 deposits-only insurance support`.
- New parity tests under `fixtures/parity/v12.20/`.
- CHANGELOG entry under `1.0.0-beta.40` (or `1.0.0-rc.0` if other
  stability gates align).

## f04720e — init oracle hardening

Adds:
- `conf_filter_bps` must be in `[MIN_CONF_FILTER_BPS=50, MAX_CONF_FILTER_BPS=1000]`.
  Previously zero (= "disabled") was accepted; now rejected.
- New `MAX_ORACLE_STALENESS_SECS = 600` upper bound on staleness.

### SDK changes required
1. Validation helper `validateInitMarketArgs` (or a new one) should
   reject `confFilterBps < 50 || > 1000` and `maxStalenessSecs > 600`
   when target is `'v12.20'`.
2. Soft-warn for `'v12.17' | 'v12.19'` callers since the wrapper would
   accept their values today but the next sync will reject.

### Migration plan
- Strict mode under v12.20 target. Permissive on older targets to
  preserve backward compatibility.

## 5229c1c — Pyth discriminator + matcher tail cap

Adds:
- Pyth `PriceUpdateV2` discriminator check. Any oracle account that
  doesn't have the exact 8-byte discriminator is rejected.
- `MAX_MATCHER_TAIL_ACCOUNTS = 32` wrapper cap on `TradeCpi` tail
  accounts. Tests using more than 32 hit a new error code.

### SDK changes required
1. Mock-oracle test fixtures must include the correct 8-byte Pyth
   discriminator prefix or their decoders need updating.
2. `encodeTradeCpi` callers that pass more than 32 tail accounts must
   chunk or fail-fast client-side. Add a soft validation in the
   `buildAccountMetas` path.

### Migration plan
- Add `validatePythPriceUpdateV2(account)` helper.
- Update mock-oracle fixtures in `test/account-decode.test.ts` with
  the discriminator prefix.

## Out-of-scope items (NOT planned for v12.20 SDK)

- Layout-level v12.19 vs v12.17 detection in `detectSlabLayout`. The
  vault offset 600 -> 616 shift documented at
  `audit-2026-04-27/v12.19-diff.md` requires layout map work. Defer
  to a dedicated layout-refactor PR when v12.20 sizes also land.
  Current consumers continue to work because SDK reads MarketConfig
  fields (which are layout-stable across the v12.17 to v12.19 sync).
