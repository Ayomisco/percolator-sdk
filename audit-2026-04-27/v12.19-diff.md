# v12.17.7 → v12.19 (d760fc4) wire-format and layout diff

Source: STAGE I borsh audit + git history of wrapper at d760fc4.

## Severity legend
- BREAKING: SDK encoders that worked on v12.17 produce wire that v12.19 wrapper rejects.
- ADDITIVE: v12.19 adds optional fields. v12.17 callers see no change.
- DROPPED: v12.19 wrapper no longer reads field; SDK extra bytes will fail trailing-byte guard.
- INTERNAL: state mutation only, no wire-format effect (e.g. F-B3, H-NEW-1).

## SLAB_LEN

| metric | v12.17 | v12.19 |
|:---|---:|---:|
| Default tier SLAB_LEN | 1_484_728 | 1_525_720 |
| Vault offset | 600 | 616 |

The +16 vault offset is the v12.19 engine layout shift. SDK's parseConfig
must branch on the detected wrapper version when populating engine-side
fields. STAGE II PHASE B will extend `detectSlabLayout` accordingly.

## STAGE II PHASE C plan

For each BREAKING/ADDITIVE entry above, encoder in PHASE C will accept
`target?: 'v12.17' | 'v12.19'` defaulting to `'v12.17'` to preserve
backward compatibility for callers built against beta.37 and earlier.
